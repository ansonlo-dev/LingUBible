🔧 語言閉包問題修復報告

✅ 問題已解決

成功修復了 AuthContext 中的語言閉包問題，現在 toast 訊息會正確反映當前選擇的語言。

🐛 問題根源

問題出現在 AuthContext.tsx 中的 React hooks 閉包問題：

  • useLanguage hook 在 AuthProvider 組件的頂層被調用
  • t 函數在組件初始化時被固定，不會隨語言變化而更新
  • 當用戶切換語言時，AuthProvider 組件不會重新渲染
  • 導致 toast 訊息仍然使用舊的語言狀態

🔍 技術分析

這是一個典型的 React hooks 閉包問題：

問題代碼：

export function AuthProvider({ children }) { const [user, setUser] = useState(null); const { t } = useLanguage(); // ❌ 在頂層調用 const login = async () => { // ... toast({ title: t('toast.loginSuccess'), // ❌ 使用舊的 t 函數 }); }; }

修復代碼：

export function AuthProvider({ children }) { const [user, setUser] = useState(null); // ✅ 移除頂層的 useLanguage 調用 const login = async () => { // ... const { t } = useLanguage(); // ✅ 在函數內部調用 toast({ title: t('toast.loginSuccess'), // ✅ 使用最新的 t 函數 }); }; }

📝 修復詳情

修復涉及以下變更：

  • 移除頂層 useLanguage：從 AuthProvider 組件頂層移除 const { t } = useLanguage();
  • 函數內部調用：在 login、register、logout 函數內部調用 useLanguage
  • 確保最新狀態：每次執行時都獲取最新的語言狀態和翻譯函數

⚠️ 為什麼會發生這個問題？

React hooks 的閉包特性導致：

  1. 初始化時固定：t 函數在組件初始化時被創建並固定
  2. 不會自動更新：語言變化時，AuthProvider 組件不會重新渲染
  3. 閉包捕獲舊值：login 函數閉包捕獲了初始化時的 t 函數
  4. 狀態不同步：即使語言狀態改變，函數內的 t 仍指向舊狀態
